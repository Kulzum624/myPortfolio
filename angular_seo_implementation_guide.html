Angular 19 SEO implementation guide for Kulzum Mujtaba's portfolio

1

Add Angular SSR (server-side rendering)

One command — makes your whole site crawlable by Google

Critical ▾

Angular 19 has SSR built-in. One command adds it. This pre-renders your HTML so Google reads all your content instead of an empty page.

ng add @angular/ssrrun in terminal

Then build with prerender (perfect for a single-page portfolio — generates static HTML files at build time):

ng build --prerenderreplaces "ng build"

Update angular.json to always prerender on build. Inside your build configuration:

📄 angular.json
"prerender": {
  "routesFile": "routes.txt"
},
"ssr": {
  "entry": "src/main.server.ts"
}
📄 routes.txt (create in project root)
/

Since you have a single-page portfolio with sections (not separate routes), just listing / is enough.

3

Use Angular's Title & Meta services in AppComponent

Updates tags dynamically — essential for SSR hydration

Critical ▾
📄 src/app/app.component.ts
import { Component, OnInit } from '@angular/core';
import { Title, Meta } from '@angular/platform-browser';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
})
export class AppComponent implements OnInit {

  constructor(
    private titleService: Title,
    private metaService: Meta,
  ) {}

  ngOnInit(): void {
    this.titleService.setTitle(
      'Kulzum Mujtaba | Frontend Developer · Angular · TypeScript · Karachi'
    );

    this.metaService.addTags([
      { name: 'description',
        content: 'Freelance frontend developer in Angular, TypeScript and Tailwind CSS. Karachi, Pakistan. Hire on Upwork and Fiverr.' },
      { name: 'keywords',
        content: 'Angular developer, TypeScript, Tailwind CSS, freelance, Upwork, Fiverr, Karachi, Pakistan, frontend developer for hire' },
      { property: 'og:title',
        content: 'Kulzum Mujtaba | Frontend Developer' },
      { property: 'og:description',
        content: 'Angular & TypeScript developer available for hire. Building fast, clean UIs.' },
    ]);
  }
}
4

Add robots.txt and sitemap.xml as static assets

Tells Google what to crawl and how often

Important ▾
📄 src/robots.txt (create this file)
User-agent: *
Allow: /
Sitemap: https://kulzummujtabafolio.netlify.app/sitemap.xml
📄 src/sitemap.xml (create this file)
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://kulzummujtabafolio.netlify.app/</loc>
    <lastmod>2025-06-22</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>

Then register both files as assets in your build config:

📄 angular.json → projects → your-app → architect → build → options → assets
"assets": [
  "src/favicon.ico",
  "src/assets",
  "src/robots.txt",
  "src/sitemap.xml"
]
5

Create an OG preview image (1200×630px)

Shows when you share your link on WhatsApp, LinkedIn, Upwork DMs

Important ▾

Make a simple 1200×630px image in Figma or Canva with your name, title, and a clean background. Save it as og-image.png and put it in src/assets/.

src/
└── assets/
    └── og-image.png   ← 1200 × 630 px

Then update the og:image and twitter:image URLs in index.html to point to /assets/og-image.png. This image appears as a card preview every time you paste your portfolio URL anywhere.

6

Fix semantic HTML — use real heading tags

Google reads H1/H2/H3 to understand your page topic

Important ▾

Check each section component and make sure headings use real HTML tags, not styled divs. This is the pattern to follow:

📄 hero component template — example
<!-- ❌ Wrong — Google ignores this -->
<div class="text-5xl font-bold">Kulzum Mujtaba</div>
<div class="text-xl">Frontend Developer</div>

<!-- ✅ Correct — same Tailwind classes, but real tags -->
<h1 class="text-5xl font-bold">Kulzum Mujtaba</h1>
<h2 class="text-xl">Frontend Developer</h2>
<!-- Section headings -->
<h2 class="...">About Me</h2>
<h2 class="...">Projects</h2>
<h2 class="...">Skills</h2>
<h2 class="...">Contact</h2>

<!-- Project titles -->
<h3 class="...">{{ project.name }}</h3>
<p>{{ project.description }}</p>
7

Submit to Google Search Console

After deploying — tell Google you exist

Final step ▾

After deploying the updated build:

1. Go to → search.google.com/search-console
2. Add property → URL prefix → paste your Netlify URL
3. Verify ownership (download HTML file → put in src/ → add to assets in angular.json → redeploy)
4. Go to Sitemaps → enter:  sitemap.xml  → Submit
5. Go to URL Inspection → paste your URL → Request Indexing

Google will crawl your site within 1–3 days. Once indexed, it starts showing in results. Check back after 1 week to see impressions.
